## Критические проблемы
1. **Блок конфигурации имеет самый низкий уровень зрелости (8%)** — это главный технический долг проекта. Дублирование конфигурации (agent_config в dev.yaml) нарушает архитектурные принципы и мешает изоляции агентов.
2. Общий кэш промптов между агентами из-за регистрации сервисов в инфраструктуре
3. Время инициализации агента слишком высокое (~1200 мс вместо целевых ≤ 100 мс)
4. Высокое потребление памяти (~4.2 ГБ для 10 агентов вместо целевых ≤ 1.2 ГБ)
5. Отсутствие валидации статусов версий для продакшена

## Рекомендации
1. Удалить agent_config и prompt_versions из dev.yaml
2. Перенести версии в data/registry.yaml
3. Исправить type_provider → provider_type во всех конфигах
4. Перенос регистрации PromptService/ContractService из InfrastructureContext в ApplicationContext
5. Реализация валидации статусов версий для профилей prod/sandbox
