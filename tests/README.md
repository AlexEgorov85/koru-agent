# Система тестирования агента

## Обзор

Этот проект реализует комплексную систему тестирования для агента, включающую:

- Поведенческие тесты (behavior tests)
- Сценарии сбоев (failure scenarios)
- Интеграционные тесты
- Архитектурные тесты

## Структура тестов

```
tests/
├── support/                    # Вспомогательные классы для тестирования
│   └── test_agent.py          # Единый интерфейс для тестирования агента
├── behavior/                   # Поведенческие тесты
│   └── agent/
│       ├── test_simple_success.py     # Простой сценарий успеха
│       ├── test_pattern_switching.py  # Смена паттернов
│       └── test_error_recovery.py     # Восстановление после ошибок
├── scenarios/
│   └── failures/              # Сценарии сбоев
│       ├── test_tool_failure.py       # Сбой инструмента
│       ├── test_recovery_failure.py   # Сбой восстановления
│       ├── test_infinite_loop.py      # Бесконечный цикл
│       └── test_llm_hallucination.py  # Галлюцинации LLM
├── unit/                      # Модульные тесты (существующие)
├── integration/               # Интеграционные тесты (существующие)
├── application/               # Старые архитектурные тесты (@legacy, @architecture_only)
├── domain/                    # Старые архитектурные тесты (@legacy, @architecture_only)
├── orchestration/             # Старые архитектурные тесты (@legacy, @architecture_only)
├── system_context/            # Старые архитектурные тесты (@legacy, @architecture_only)
├── fixtures/                  # Фикстуры для тестов (существующие)
├── temp_support/              # Временные вспомогательные тестовые скрипты (не часть основной структуры)
├── debug/                     # Отладочные тесты и скрипты
└── conftest.py               # Конфигурация pytest с маркерами
```

## Основные компоненты

### TestAgent
Единый black-box интерфейс для тестирования агента. Обеспечивает:

- Детерминированный LLM для воспроизводимых тестов
- Контролируемые инструменты с заданным поведением
- Отслеживание событий и состояний
- Механизмы восстановления

### Типы тестов

#### Поведенческие тесты (`@behavior`)
- Тестируют нормальное поведение агента
- Проверяют сценарии использования
- Убеждаются, что агент достигает целей

#### Тесты сбоев (`@failure`)
- Тестируют реакцию на различные сбои
- Проверяют механизмы восстановления
- Убеждаются, что система устойчива

## Маркеры pytest

- `@pytest.mark.behavior` - поведенческие тесты
- `@pytest.mark.failure` - тесты сбоев
- `@pytest.mark.legacy` - устаревшие тесты
- `@pytest.mark.architecture_only` - архитектурные тесты

## Запуск тестов

```bash
# Запуск всех тестов
python -m pytest

# Запуск только поведенческих тестов
python -m pytest -m behavior

# Запуск только тестов сбоев
python -m pytest -m failure

# Запуск конкретного теста
python -m pytest tests/behavior/agent/test_simple_success.py
```

## Принципы тестирования

1. **Детерминированность**: все тесты должны быть воспроизводимы
2. **Изоляция**: каждый тест должен быть независим
3. **Покрытие**: тесты должны покрывать все сценарии использования
4. **Устойчивость**: тесты должны быть устойчивы к изменениям в реализации
