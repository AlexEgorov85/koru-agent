# Система тестирования

## Структура тестов

В проекте реализована многоуровневая система тестирования, соответствующая новой архитектуре с изолированными контекстами:

```
tests/
├── unit/                    # Модульные тесты
│   ├── core_infrastructure/ # Тесты инфраструктурных компонентов
│   └── ...                  # Другие модульные тесты
├── integration/             # Интеграционные тесты
│   └── infrastructure/      # Интеграционные тесты инфраструктуры
├── e2e/                     # Сквозные тесты
└── verified/                # Проверенные функциональные тесты
    ├── test_module_reload_functionality.py    # Тест перезагрузки модулей
    ├── test_planning_skill.py                 # Тест навыка планирования
    └── test_strategy_support.py               # Тест поддержки стратегий
```

## Архитектура тестирования

Тесты соответствуют новой архитектуре приложения:

- **Изолированные контексты**: каждый тест использует изолированный `ApplicationContext`
- **Конфигурация через ComponentConfig**: все тесты используют `ComponentConfig` вместо устаревшего `AgentConfig`
- **Изоляция ресурсов**: каждый контекст имеет свои изолированные кэши промптов и контрактов
- **Поддержка профилей**: тесты могут работать в `prod` и `sandbox` режимах

## Текущее состояние

Система тестирования полностью обновлена и включает:

1. **Модульные тесты** - проверяют отдельные компоненты
2. **Интеграционные тесты** - проверяют взаимодействие между компонентами
3. **Сквозные тесты** - проверяют полные сценарии использования
4. **Функциональные тесты** - проверяют бизнес-логику

Все тесты проходят успешно без предупреждений.

## Запуск тестов

Для запуска всех тестов:

```bash
python -m pytest tests/ -v
```

Для запуска тестов определенного уровня:

```bash
# Модульные тесты
python -m pytest tests/unit/ -v

# Интеграционные тесты
python -m pytest tests/integration/ -v

# Конкретный тест
python -m pytest tests/verified/test_planning_skill.py -v
```

## Особенности новой архитектуры

- Используется `ComponentConfig` для конфигурации контекстов
- Каждый `ApplicationContext` имеет изолированные кэши
- Поддерживается горячая замена версий через `clone_with_version_override`
- Реализована изоляция между продакшн и песочницей
- Обеспечена безопасность через ограничение побочных эффектов
