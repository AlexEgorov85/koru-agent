# Интеграционные тесты агента

Эта директория содержит интеграционные тесты для проверки полного жизненного цикла агента с реальными промтами и детерминированным поведением.

## Структура

- `test_full_execution_lifecycle.py` - Основной интеграционный тест полного жизненного цикла агента
- `fixtures/` - Вспомогательные файлы для тестов
  - `test_project.py` - Создание тестовой структуры проекта
  - `llm_scenarios.py` - Сценарии ответов LLM для детерминированного поведения

## Тестируемые компоненты

Интеграционный тест проверяет интеграцию следующих компонентов:

1. **Загрузка реальных промтов** из файловой системы (`prompts/`)
2. **Формирование промта** с подстановкой переменных
3. **Вызов мока LLM** с детерминированным ответом
4. **Парсинг ответа** и выполнение инструментов
5. **Обновление состояния** и снапшотов агента
6. **События** публикуются через шину событий

## Особенности теста

- **Реалистичный сценарий**: Анализ структуры проекта и поиск Python-файлов
- **Изоляция**: Никаких внешних зависимостей (только файлы промтов + моки)
- **Детерминированность**: Ответы LLM предопределены для воспроизводимости
- **Воспроизводимость**: Временные директории для изоляции тестовых данных
- **Автоматическая очистка**: После теста все временные данные удаляются

## Проверки

Тест выполняет следующие проверки:

- [x] Промт загружен из реального файла (не мок)
- [x] Переменные промта валидированы по схеме
- [x] Снапшот создан до вызова LLM с подставленными переменными
- [x] Ответ LLM детерминирован и соответствует ожидаемой структуре
- [x] Снапшот обновлён после получения ответа
- [x] Инструменты выполнены с правильными параметрами
- [x] Состояние агента корректно обновлено (шаги, прогресс, ошибки)
- [x] События опубликованы через шину (инициализация, шаг, завершение)

## Запуск тестов

```bash
# Запуск конкретного интеграционного теста
pytest tests/integration/agent/test_full_execution_lifecycle.py::TestAgentFullExecutionLifecycle::test_agent_analyzes_project_structure_with_real_prompts -v

# Запуск всех интеграционных тестов агента
pytest tests/integration/agent/ -v
```

## Назначение

Этот тест служит "канарейкой" для системы - если он падает, значит сломана интеграция компонентов, а не бизнес-логика отдельного модуля.
