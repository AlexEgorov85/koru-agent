# Система бенчмаркинга для агентов

## Обзор

Этот проект представляет собой систему бенчмаркинга для агентов, которая позволяет оценивать и сравнивать производительность различных агентов на наборе стандартизированных задач. Система поддерживает:

- Стандартные бенчмарки для оценки производительности агентов
- Гибкие метрики оценки (точность, скорость, полнота решения и др.)
- Интеграцию с различными моделями ИИ
- Автоматизированное выполнение тестов
- Детализированные отчеты о результатах

## 🧪 Философия тестирования

В проекте принята строгая философия тестирования, направленная на обеспечение высокого качества и доверия к системе:

### 🔸 Разрешенные моки
- **LLM Mock** — эмуляция ответа языковой модели (`TestLLMProvider`)
- **Database Mock** — in-memory / fake DB provider (`InMemoryDBProvider`)

### 🔸 Запрещенные практики
- Мокать сервисы application-слоя
- Мокать domain-объекты
- Мокать репозитории (кроме DB boundary)
- Мокать capability / prompt / renderer / analyzer / benchmark runner
- Использовать `Mock()` или `AsyncMock()` для внутренних компонентов

### 🔸 Стратегия тестирования
1. **Domain tests (100% без моков)** — тестирование чистой бизнес-логики
2. **Application tests (только централизованные моки)** — тестирование прикладных сервисов
3. **Integration tests (end-to-end)** — тестирование полных сценариев

### 🔸 Критерии качества
- Невозможно написать тест, замокав `PromptRenderer`
- Невозможно написать тест, замокав `Capability`
- Любой реальный сценарий агента воспроизводится в тесте
- Тесты ловят ошибки конфигурации, несовместимость, валидацию и регрессии

Для подробностей см. файл `TESTING_PHILOSOPHY.md`.
