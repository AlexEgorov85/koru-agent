# ADR-0002: Контрактная валидация компонентов через манифесты

**Дата:** 2026-02-19  
**Статус:** Принято  
**Авторы:** Agent_v5 Team

---

## Контекст

При разработке системы возникла проблема:
- Компоненты не имели явных контрактов
- Валидация выполнялась во время выполнения
- Ошибки обнаруживались слишком поздно
- Отсутствие версионирования контрактов

## Решение

Внедрить контрактную валидацию через манифесты:

### Manifest Structure
```yaml
component_id: skill.planning
version: v1.0.0
input_contracts:
  planning.create_plan: v1.0.0
output_contracts:
  planning.create_plan: v1.0.0
prompt_versions:
  planning.create_plan: v1.0.0
```

### Валидация при инициализации
1. Загрузка манифеста из файловой системы
2. Проверка наличия всех контрактов
3. Проверка совместимости версий
4. Кэширование валидированных ресурсов

## Последствия

### Положительные
- Раннее обнаружение ошибок (fail-fast)
- Явные зависимости между компонентами
- Возможность версионирования API
- Автоматическая документация

### Отрицательные
- Усложнение процесса добавления компонентов
- Необходимость поддержки контрактов
- Дополнительная валидация при запуске
