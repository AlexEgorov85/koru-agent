# Production Deployment Checklist для PromptRepository

## 1. Подготовка к миграции

### 1.1 Проверка совместимости

- [x] Убедитесь, что все существующие промты конвертированы в формат PromptVersion
- [x] Проверьте, что все переменные в промтах определены в схеме variables_schema
- [x] Убедитесь, что все шаблоны используют формат `{{variable_name}}` для подстановки
- [x] Проверьте, что все навыки и агенты используют Capability с указанием версий промтов

### 1.2 Обновление зависимостей

- [x] Установите все новые зависимости для PromptRepository
- [x] Обновите версии существующих зависимостей при необходимости
- [x] Проверьте совместимость с GreenPlum/PostgreSQL

### 1.3 Создание резервной копии

- [x] Создайте резервную копию текущей системы промтов
- [x] Зафиксируйте текущее состояние в Git

## 2. Деплой компонентов

### 2.1 Модель данных

- [x] Обновите модель `PromptVersion` до последней версии с поддержкой:
  - Semantic versioning (MAJOR.MINOR.PATCH)
  - Доменной адресации (domain:provider:capability:role)
  - Схемы переменных с валидацией
  - Контракта ожидаемого ответа
  - Жизненного цикла (draft → active → shadow → deprecated → archived)
  - Метрик использования

### 2.2 Репозиторий

- [x] Установите `DatabasePromptRepository` как основной репозиторий
- [x] Настройте `CachedPromptRepository` поверх базы данных
- [x] Проверьте, что кэш корректно инвалидируется при изменениях

### 2.3 Рендеринг

- [x] Установите `PromptRenderer` с поддержкой валидации переменных
- [x] Проверьте, что все подстановки переменных защищены от инъекций
- [x] Убедитесь, что валидация происходит до вызова LLM

### 2.4 Инициализация

- [x] Настройте `PromptSystemInitializer` для загрузки промтов из файлов в БД
- [x] Убедитесь, что синхронизация файлов и БД работает корректно

## 4. Проверка функциональности

### 4.1 Жизненный цикл промтов

- [ ] Создайте новый промт в статусе `draft`
- [ ] Активируйте промт и проверьте, что он становится активным
- [ ] Создайте теневую (shadow) версию для A/B тестирования
- [ ] Депрекейтите промт и проверьте, что он больше не используется
- [ ] Архивируйте промт и проверьте, что он не доступен

### 4.2 Валидация переменных

- [ ] Проверьте, что обязательные переменные обязательны к предоставлению
- [ ] Убедитесь, что типы переменных проверяются корректно
- [ ] Проверьте, что промты не рендерятся при ошибках валидации
- [ ] Убедитесь, что ошибки валидации логируются корректно

### 4.3 Снапшоты выполнения

- [ ] Убедитесь, что все вызовы промтов создают снапшоты
- [ ] Проверьте, что снапшоты содержат всю необходимую информацию
- [ ] Убедитесь, что ошибки выполнения фиксируются в снапшотах
- [ ] Проверьте, что метрики обновляются на основе снапшотов

### 4.4 Работа с файловой системой

- [ ] Проверьте, что новые промты из файлов синхронизируются в БД
- [ ] Убедитесь, что изменения в файлах обновляют БД
- [ ] Проверьте, что версии в файлах соответствуют семантическому версионированию
- [ ] Убедитесь, что формат файлов соответствует требованиям

## 5. Мониторинг и логирование

### 5.1 Метрики

- [ ] Убедитесь, что собираются метрики использования промтов
- [ ] Проверьте, что отслеживается процент ошибок
- [ ] Убедитесь, что фиксируется время генерации
- [ ] Проверьте, что отслеживается процент отклонений валидатором

### 5.2 Логирование

- [ ] Убедитесь, что все вызовы LLM логируются
- [ ] Проверьте, что ошибки валидации переменных логируются
- [ ] Убедитесь, что снапшоты выполнения сохраняются
- [ ] Проверьте, что все изменения статусов промтов логируются

## 6. Обратная совместимость

### 6.1 Старые навыки

- [ ] Убедитесь, что старые навыки работают с новым PromptRepository
- [ ] Проверьте, что Capability могут использовать старый формат (если нужно)
- [ ] Убедитесь, что нет ошибок при отсутствии привязки к версиям промтов

### 6.2 Старые промты

- [ ] Проверьте, что старые промты корректно мигрированы
- [ ] Убедитесь, что старые шаблоны продолжают работать
- [ ] Проверьте, что старые переменные подставляются корректно

## 7. Production готовность

### 7.1 Безопасность

- [ ] Убедитесь, что все подстановки переменных защищены от инъекций
- [ ] Проверьте, что только определенные в схеме переменные подставляются
- [ ] Убедитесь, что чувствительные данные не попадают в логи
- [ ] Проверьте, что промты не могут содержать вредоносные инструкции

### 7.2 Производительность

- [ ] Убедитесь, что кэширование снижает время доступа к промтам
- [ ] Проверьте, что БД не становится узким местом
- [ ] Убедитесь, что снапшоты не влияют на производительность
- [ ] Проверьте, что валидация не замедляет выполнение

### 7.3 Надежность

- [ ] Убедитесь, что система может работать без файловой синхронизации
- [ ] Проверьте, что ошибки валидации не прерывают выполнение
- [ ] Убедитесь, что сбои в сохранении снапшотов не влияют на основной поток
- [ ] Проверьте, что есть fallback при ошибках репозитория
