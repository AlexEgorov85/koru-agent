# Анализ папки logs

## Текущая структура (март 2026)

```
logs/
├── agent.log                    # ← СТАРОЕ (от main.py logging.basicConfig)
├── active/                      # ← СТАРАЯ СИСТЕМА (log_config.py)
│   ├── sessions/
│   │   └── latest.log           # Ссылка на последнюю сессию
│   └── llm/
├── archive/                     # ← СТАРАЯ СИСТЕМА (архив по месяцам)
│   └── 2026/
│       └── 02/
├── config/                      # ← СТАРАЯ СИСТЕМА (конфигурация)
│   └── logging_config.yaml
├── indexed/                     # ← СТАРАЯ СИСТЕМА (индексы для поиска)
│   ├── sessions_index.jsonl
│   └── agents_index.jsonl
├── llm_calls/                   # ← СТАРАЯ СИСТЕМА (LLM вызовы)
└── sessions/                    # ← СТАРАЯ СИСТЕМА (сессии по ID)
    ├── 015135fc-....log         # Файлы сессий (по одному на сессию)
    ├── 086545b2-....log
    └── ...
```

## Что куда пишется

### 1. **agent.log** (корень logs/)
- **Кто пишет:** `main.py` → `logging.basicConfig()`
- **Что пишет:** Общие логи приложения
- **Формат:** Текст с временем и уровнем
- **Статус:** ⚠️ **УСТАРЕЛ** - дублирует новую систему

### 2. **active/, archive/, config/, indexed/, llm_calls/**
- **Кто пишет:** Старая система логирования (`log_config.py`, `log_search.py`)
- **Что пишет:** 
  - `active/sessions/` - активные сессии
  - `archive/` - архив по месяцам
  - `config/` - конфигурация
  - `indexed/` - индексы для поиска
  - `llm_calls/` - LLM вызовы
- **Статус:** ⚠️ **НЕ ИСПОЛЬЗУЕТСЯ** новой системой

### 3. **sessions/*.log**
- **Кто пишет:** Старая система (SessionLogger)
- **Что пишет:** Логи по сессиям (один файл = одна сессия)
- **Статус:** ⚠️ **НЕ ИСПОЛЬЗУЕТСЯ** новой системой

## Новая система (handlers.py)

### Куда пишет новая система:

```python
# В core/infrastructure/context/infrastructure_context.py
log_config = LoggingConfig(
    file=FileOutputConfig(
        enabled=True,
        level=LogLevel.DEBUG,
        format=LogFormat.JSONL,
        base_dir=Path("logs"),           # ← Базовая директория
        organize_by_session=True,        # ← Организация по сессиям
        organize_by_date=True,           # ← Организация по датам
    )
)
```

### Планируемая структура новой системы:

```
logs/
├── sessions/                    # ← Новая система (по сессиям)
│   ├── {session_id}/

│   └── {session_id}/

├── common/                      # ← Новая система (общие логи)
│   └── 2026-03-03_common.log
└── by_date/                     # ← Новая система (по датам)
    └── 2026-03-03.log
```

**НО!** Сейчас новая система **НЕ АКТИВНА** - обработчики создаются, но события не записываются.

## Проблемы

### 1. Две параллельные системы

| Характеристика | Старая система | Новая система |
|---------------|----------------|---------------|
| Файлы конфигурации | `log_config.py` | `config.py` |
| Обработчики | SessionLogger | `TerminalLogHandler`, `FileLogHandler` |
| Формат | Текст | JSONL |
| Организация | Один файл на сессию | По сессиям + датам |
| Статус | ✅ Работает | ⚠️ Частично работает |

### 2. Старые папки не используются новой системой

**Можно удалить:**
- `logs/active/` - старая система
- `logs/archive/` - старая система
- `logs/config/` - старая система
- `logs/indexed/` - старая система
- `logs/llm_calls/` - старая система
- `logs/sessions/*.log` - старая система (плоская структура)
- `logs/agent.log` - дублирует новую систему

**Нужно сохранить:**
- Ничего из старого - новая система создаст свои папки

### 3. Разделение на сессии

**Старая система:**
- Один файл на сессию: `sessions/{session_id}.log`
- Все шаги сессии в одном файле

**Новая система (планируется):**
- По сессиям И датам: `sessions/{session_id}/{date}_session.log`
- Один файл в день на сессию
- Автоматическая ротация по размеру (100MB)

## Рекомендации

### 1. Включить новую систему полностью

Сейчас `FileLogHandler` создается в `InfrastructureContext`, но нужно проверить что:
1. ✅ Обработчик подписан на события
2. ✅ События действительно записываются в файлы

### 2. Отключить старую систему

В `InfrastructureContext` удалить/закомментировать:
```python
# Удалить:
self.log_storage = FileSystemLogStorage(logs_dir)  # Старая система
```

### 3. Очистить папку logs

После тестирования новой системы:
```bash
# Удалить старые папки
rmdir /s /q logs\active
rmdir /s /q logs\archive
rmdir /s /q logs\config
rmdir /s /q logs\indexed
rmdir /s /q logs\llm_calls
del logs\sessions\*.log
del logs\agent.log
```

### 4. Где искать информацию

| Что ищем | Где (старая) | Где (новая) |
|----------|--------------|-------------|
| Логи сессии | `logs/sessions/{id}.log` | `logs/sessions/{id}/{date}_session.log` |
| LLM вызовы | `logs/llm_calls/` | В файлах сессии (JSONL) |
| Поиск по логам | `logs/indexed/` | Через `log_search.py` |
| Архив | `logs/archive/{year}/{month}/` | Автоматическая ротация |

## Выводы

1. **Старая система** создает много папок, но **не используется** новым кодом
2. **Новая система** готова, но требует включения и тестирования
3. **Разделение на сессии** есть в обеих системах, но по-разному реализовано
4. **Можно безопасно удалить** старые папки после проверки новой системы
