# Роли промтов

Роли промтов в Koru AI Agent Framework определяют функциональное назначение каждого промта в диалоге с LLM. Правильное использование ролей позволяет структурировать взаимодействие с LLM, обеспечивая предсказуемость и согласованность поведения агентов.

## Определение ролей

Каждый промт в системе имеет назначенную роль, которая определяет его место и функцию в цепочке взаимодействия с LLM. Роли соответствуют общепринятым шаблонам взаимодействия с LLM и поддерживаются большинством современных провайдеров.

## Типы ролей

### 1. System (Системный)

Системные промты определяют общее поведение и контекст для LLM. Они устанавливают основные правила, роли и ограничения для всего взаимодействия.

#### Назначение:
- Задание основной роли агента
- Установка контекста выполнения задачи
- Определение правил поведения
- Задание ограничений и требований

#### Пример использования:
```markdown
---
provider: openai
role: system
status: active
---

# Инструкции для агента анализа безопасности кода

Ты являешься экспертом в области безопасности кода. При анализе кода следуй следующим принципам:

1. Идентифицируй потенциальные уязвимости в коде
2. Оцени уровень риска каждой уязвимости
3. Предложи конкретные рекомендации по устранению
4. Объясни, почему каждая уязвимость представляет риск

Ты должен отвечать только в формате JSON с определенной структурой.
```

#### Когда использовать:
- При инициализации агента
- Для задания основных правил поведения
- Для определения доменной области
- Для установки ограничений и требований

### 2. User (Пользовательский)

Пользовательские промты содержат фактический ввод от пользователя или задачу, которую нужно решить. Они представляют собой конкретные запросы или данные для обработки.

#### Назначение:
- Предоставление конкретной задачи или запроса
- Передача входных данных для обработки
- Задание контекста для текущего шага
- Формулирование вопросов или проблем

#### Пример использования:
```markdown
---
provider: openai
role: user
status: active
variables:
  - name: code_snippet
    type: string
    required: true
    description: "Фрагмент кода для анализа"
---

Проанализируй следующий фрагмент кода на наличие уязвимостей:

```
{{code_snippet}}
```
```

#### Когда использовать:
- Для передачи пользовательского ввода
- Для предоставления данных для обработки
- Для формулирования конкретных задач
- Для задания контекста текущего шага

### 3. Assistant (Ассистент)

Промты ассистента эмулируют ответы модели и используются для предоставления примеров правильного поведения или для few-shot обучения.

#### Назначение:
- Предоставление примеров правильных ответов
- few-shot обучение
- Демонстрация ожидаемого формата
- Корректировка поведения модели

#### Пример использования:
```markdown
---
provider: openai
role: assistant
status: active
---

{
  "vulnerabilities": [
    {
      "type": "SQL Injection",
      "severity": "HIGH",
      "location": {
        "line": 15,
        "column": 10
      },
      "description": "Potential SQL injection vulnerability in user input handling",
      "recommendation": "Use parameterized queries to prevent injection attacks"
    }
  ],
  "overall_risk": "HIGH"
}
```

#### Когда использовать:
- Для предоставления примеров ответов
- Для few-shot обучения
- Для демонстрации формата вывода
- Для корректировки поведения модели

### 4. Tool (Инструмент)

Промты инструментов используются для взаимодействия с внешними инструментами и вызовов функций.

#### Назначение:
- Форматирование вызовов инструментов
- Определение структуры вызовов функций
- Обработка результатов инструментов
- Интеграция с внешними системами

#### Пример использования:
```markdown
---
provider: openai
role: tool
status: active
variables:
  - name: tool_name
    type: string
    required: true
    description: "Название инструмента"
  - name: arguments
    type: object
    required: true
    description: "Аргументы вызова инструмента"
expected_response:
  type: object
  properties:
    result:
      type: object
      description: "Результат выполнения инструмента"
    error:
      type: string
      description: "Описание ошибки, если произошла"
---

{"name": "{{tool_name}}", "arguments": {{arguments}}}
```

#### Когда использовать:
- Для форматирования вызовов инструментов
- Для определения структуры вызовов функций
- Для обработки результатов инструментов
- Для интеграции с внешними системами

## Иерархия ролей

Роли организованы в иерархию в соответствии с их влиянием на поведение модели:

### 1. System (наивысший приоритет)
- Устанавливает основной контекст
- Определяет общее поведение
- Редко изменяется в процессе сессии

### 2. User (текущий запрос)
- Предоставляет конкретный запрос
- Может изменяться на каждом шаге
- Влияет на текущий контекст

### 3. Assistant (ответ модели)
- Представляет собой ответ модели
- Может использоваться для коррекции
- Влияет на будущие ответы

### 4. Tool (вспомогательная роль)
- Обслуживает основные роли
- Обычно используется вспомогательно
- Не определяет основное поведение

## Использование ролей в цепочке

Роли используются в следующем порядке в типичной цепочке взаимодействия:

```
[System Prompt] -> [User Prompt] -> [Assistant Response] -> [Tool Calls]
```

### Пример полной цепочки:

1. **System**: Установка роли агента анализа безопасности
2. **User**: Предоставление кода для анализа
3. **Assistant**: Ответ с результатами анализа
4. **Tool**: Вызов инструмента для дополнительной проверки

## Лучшие практики

### 1. Ясность и конкретность
- Системные промты должны быть ясными и недвусмысленными
- Пользовательские промты должны точно описывать задачу
- Ассистентские промты должны демонстрировать ожидаемое поведение

### 2. Согласованность
- Используйте согласованные роли в пределах одного домена
- Поддерживайте единообразие в формулировках
- Следуйте установленным шаблонам

### 3. Адаптивность
- Адаптируйте роли под конкретную задачу
- Используйте переменные для гибкости
- Обеспечьте возможность переиспользования

### 4. Тестирование
- Тестируйте промты разных ролей отдельно
- Проверяйте взаимодействие между ролями
- Мониторьте эффективность каждой роли

## Интеграция с агентами

Роли интегрированы с агентами следующим образом:

- **Системные промты** устанавливают базовое поведение агента
- **Пользовательские промты** передают задачи агенту
- **Ассистентские промты** используются для коррекции поведения
- **Инструментальные промты** обеспечивают интеграцию с инструментами

## Структура в файловой системе

Роли отражаются в структуре файловой системы:

```
prompts/
├── {domain}/
│   └── {capability}/
│       ├── system/     # Системные промты
│       │   └── v{version}.md
│       ├── user/       # Пользовательские промты
│       │   └── v{version}.md
│       ├── assistant/  # Промты ассистента
│       │   └── v{version}.md
│       ├── tool/       # Инструментальные промты
│       │   └── v{version}.md
│       └── _index.yaml
```

## Валидация ролей

Система включает валидацию ролей:

- Проверка корректности назначенной роли
- Проверка соответствия содержимого роли
- Проверка совместимости ролей в цепочке
- Проверка уникальности ролей в пределах версии

## Расширение ролей

При необходимости система позволяет расширять набор ролей, добавляя новые специфичные роли для конкретных сценариев использования, при этом сохраняя совместимость с основными ролями.
