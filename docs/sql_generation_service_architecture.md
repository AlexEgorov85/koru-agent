# Архитектура сервиса генерации и коррекции SQL

## Обзор

SQLGenerationService - это централизованный сервис для безопасной генерации и коррекции SQL-запросов, который решает следующие задачи:

1. **Предотвращение SQL-инъекций** через обязательную параметризацию запросов
2. **Централизованная генерация** запросов для всех навыков
3. **Автоматическая коррекция** ошибок выполнения запросов
4. **Валидация безопасности** с оценкой рисков

## Архитектура

### Компоненты

```
core/
└── infrastructure/
    └── service/
        ├── sql_generation/          # Модуль генерации SQL
        │   ├── service.py           # SQLGenerationService (фасад)
        │   ├── validator.py         # SQLQueryValidator
        │   ├── error_analyzer.py    # SQLErrorAnalyzer
        │   ├── correction.py        # SQLCorrectionEngine
        │   ├── schema.py            # Pydantic схемы
        │   └── __init__.py          # Пакет модуля
        └── base_service.py          # Базовый класс сервиса
```

### Основные классы

#### SQLGenerationService
- **Назначение**: Главный фасад сервиса
- **Функции**:
  - Генерация безопасных SQL-запросов
  - Автоматическая коррекция ошибок
  - Интеграция с другими сервисами
  - Публикация событий через EventBus

#### SQLQueryValidator
- **Назначение**: Валидация безопасности SQL-запросов
- **Функции**:
  - Проверка разрешенных операций (только SELECT/WITH)
  - Валидация параметризации
  - Проверка имен таблиц/колонок
  - Расчет оценки безопасности

#### SQLErrorAnalyzer
- **Назначение**: Анализ ошибок выполнения SQL-запросов
- **Функции**:
  - Классификация типа ошибки
  - Извлечение информации о проблемных таблицах
  - Формирование рекомендаций по исправлению

#### SQLCorrectionEngine
- **Назначение**: Коррекция SQL-запросов на основе ошибок
- **Функции**:
  - Применение стратегий коррекции
  - Генерация исправленного запроса
  - Оценка уверенности в исправлении

## Безопасность

### Защита от SQL-инъекций
- Обязательная параметризация всех запросов
- Валидация через SQLQueryValidator
- Белый список разрешенных операций (SELECT, WITH)
- Санитизация имен таблиц и колонок

### Ограничения
- Только операции чтения (SELECT, WITH)
- Ограничение на вложенные запросы
- Контроль количества возвращаемых строк

## Использование

### В навыках

```python
# В методе навыка
sql_service = self.system_context.get_service("sql_generation_service")

generation_input = SQLGenerationInput(
    user_question="Какие книги написал Толстой?",
    tables=["books", "authors"],
    max_rows=10
)

result = await sql_service.execute_with_auto_correction(
    generation_input,
    context=session_context
)
```

### Схемы данных

#### SQLGenerationInput
- `user_question`: Вопрос пользователя
- `tables`: Список таблиц для использования
- `context`: Дополнительный контекст
- `max_rows`: Максимальное количество строк

#### SQLGenerationOutput
- `sql`: Сгенерированный SQL-запрос
- `reasoning`: Обоснование выбора запроса
- `tables_used`: Использованные таблицы
- `parameters`: Параметры для параметризованного запроса

## Интеграция с системой

### Регистрация сервиса
Сервис автоматически регистрируется в SystemContext при инициализации системы.

### Зависимости
- PromptService: для генерации через LLM
- TableDescriptionService: для получения метаданных таблиц
- EventBus: для публикации событий

## Автоматическая коррекция

Сервис поддерживает автоматическую коррекцию ошибок выполнения:
1. Выполнение запроса
2. При ошибке - анализ через SQLErrorAnalyzer
3. Коррекция через SQLCorrectionEngine
4. Повторное выполнение (до 3 попыток)

## События

Сервис публикует следующие события через EventBus:
- `sql_generation.generation_success/failure`
- `sql_correction.correction_success/failure`
- `sql_execution.execution_success/failure`
