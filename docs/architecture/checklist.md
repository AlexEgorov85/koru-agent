1. **Каждые 2 недели** заполняйте статусы по всем пунктам
2. **Фиксируйте тренд** по сравнению с предыдущей проверкой:

   - 📉 Если ухудшилось (было ✅ → стало ⚠️)

3. **Обновляйте дату** последней проверки
4. **Добавляйте комментарии** для контекста (особенно при ❌)
5. **Пересчитывайте итоги** в сводной таблице
6. **Формируйте план** на следующие 2 недели на основе приоритетных задач

## 📌 Почему это критически важно

Дублирование конфигурации (`agent_config` в `dev.yaml`) **блокирует все остальные улучшения**:

| Проблема | Последствие | Решение |
|----------|-------------|---------|
| Версии промптов в `dev.yaml` | Невозможно горячее переключение | Перенос версий в `data/registry.yaml` |
| `agent_config` в инфраструктуре | Смешение ответственностей | Удаление, замена на `AppConfig` |
| `type_provider` вместо `provider_type` | Провайдеры не регистрируются | Исправление конфигурации |

**Без исправления конфигурации изоляция агентов невозможна** — все агенты будут использовать одни и те же версии из `dev.yaml`, а не изолированные `AppConfig`.
