# Обзор архитектуры системы

Composable AI Agent Framework реализует **чистую слоистую архитектуру** с сильным разделением ответственности. Архитектура построена в соответствии с принципами SOLID и DDD, обеспечивая гибкость, тестируемость и масштабируемость.

## Основные принципы

- **Dependency Inversion** - зависимости направлены к абстракциям, а не к конкретным реализациям
- **Event-driven коммуникация** - взаимодействие между компонентами осуществляется через события
- **Асинхронная модель выполнения** - все операции выполняются асинхронно для обеспечения высокой производительности
- **Расширяемость через плагины** - навыки, инструменты и провайдеры легко расширяются
- **Минимальная связность** - слабая связанность между компонентами для лучшей тестируемости и поддержки

## Архитектурные слои

### 1. Domain (Предметная область)

Слой предметной области содержит бизнес-логику и правила, не зависящие от внешних факторов.

**Основные компоненты:**
- **Модели предметной области** - основные сущности (Capability, Agent, Prompt и т.д.)
- **Абстракции** - интерфейсы для основных компонентов системы
- **Значения (Value Objects)** - неизменяемые объекты семантическим значением
- **Паттерны мышления** - абстракции для различных стратегий решения задач

```
domain/
├── abstractions/           # Интерфейсы и абстракции
│   ├── event_system.py     # Система событий
│   ├── prompt_repository.py # Репозиторий промтов
│   └── thinking_pattern.py  # Паттерны мышления
├── models/                 # Модели предметной области
│   ├── agent/             # Модели агентов
│   ├── benchmark/         # Модели бенчмарков
│   ├── prompt/            # Модели промтов
│   └── execution/         # Модели выполнения
└── value_objects/         # Значения (Value Objects)
    ├── domain_type.py     # Типы доменов
    └── provider_type.py   # Типы провайдеров
```

### 2. Application (Прикладная логика)

Слой прикладной логики координирует работу компонентов предметной области для выполнения пользовательских сценариев.

**Основные компоненты:**
- **Менеджеры жизненного цикла** - управление состоянием и выполнением агентов
- **Управление доменами** - адаптация агентов к различным областям задач
- **Службы промтов** - загрузка, валидация и управление версиями промтов

```
application/
├── agent/                  # Компоненты агента
│   ├── pattern_recovery_manager.py # Менеджер восстановления паттернов
│   └── pattern_state_manager.py   # Менеджер состояния паттернов
├── orchestration/          # Оркестрация выполнения
│   ├── atomic_actions/     # Атомарные действия
│   └── executors/          # Исполнители
├── services/               # Прикладные службы
│   ├── prompt_loader.py    # Загрузчик промтов
│   └── system_initialization_service.py # Инициализация системы
└── factories/              # Фабрики
    └── agent_factory.py    # Фабрика агентов
```

### 3. Infrastructure (Инфраструктура)

Слой инфраструктуры содержит реализации внешних зависимостей и технических компонентов.

**Основные компоненты:**
- **Репозитории** - реализации абстракций хранения данных
- **Провайдеры** - реализации взаимодействия с внешними системами (LLM, БД)
- **Адаптеры** - адаптация внешних интерфейсов к внутренним абстракциям
- **Инструменты** - реализации различных утилитарных возможностей

```
infrastructure/
├── repositories/           # Реализации репозиториев
│   ├── in_memory_prompt_repository.py # In-memory реализация
│   └── benchmark_repository_impl.py   # Реализация репозитория бенчмарков
├── gateways/               # Шлюзы для внешних систем
│   ├── llm_providers/      # Провайдеры LLM
│   └── database_providers/ # Провайдеры баз данных
├── adapters/               # Адаптеры
│   └── event_publisher_adapter.py # Адаптер публикации событий
├── tools/                  # Инструменты
│   ├── file_reader_tool.py # Чтение файлов
│   ├── sql_tool.py         # SQL инструмент
│   └── ast_parser_tool.py  # Парсер AST
└── services/               # Инфраструктурные службы
    └── text_similarity_service.py # Сервис текстовой схожести
```

## Компоненты системы

### Компонуемые агенты

Компонуемые агенты - это основные исполнители логики в системе. Они могут комбинировать различные паттерны мышления для решения сложных задач.

**Основные возможности:**
- Выполнение атомарных действий
- Выполнение компонуемых паттернов
- Адаптация к различным доменам
- Работа в разных стратегиях решения задач

### Паттерны мышления

Паттерны мышления - это стратегии решения задач, которые могут быть скомбинированы для решения сложных проблем. Каждый паттерн мышления определяет конкретный способ обработки информации и принятия решений.

### Управление промтами

Система управления промтами обеспечивает:
- Версионирование промтов
- Валидацию переменных и ответов
- Поддержку различных ролей (system, user, assistant)
- Аудит и метрики использования

## Поток выполнения

1. **Инициализация** - создание агента и загрузка необходимых паттернов мышления
2. **Адаптация** - адаптация агента к конкретной задаче и домену
3. **Планирование** - разработка стратегии решения задачи
4. **Выполнение** - выполнение атомарных действий и паттернов мышления
5. **Валидация** - проверка решений, принятых LLM
6. **Контроль состояния** - мониторинг прогресса и восстановление при ошибках

## Преимущества архитектуры

- **Масштабируемость** - возможность легко добавлять новые функции без изменения существующего кода
- **Тестируемость** - благодаря слабой связанности и использованию абстракций
- **Поддерживаемость** - четкое разделение ответственности между слоями
- **Гибкость** - возможность замены реализаций без изменения архитектуры
- **Контролируемость** - строгая валидация решений, принятых LLM
