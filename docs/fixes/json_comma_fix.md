# Исправление ошибок парсера JSON

## Проблемы

LLM часто генерирует невалидный JSON с несколькими типичными проблемами:

### 1. Отсутствующие запятые между полями

```json
{
  "field1": "value1"
  "field2": "value2"
}
```

### 2. Отсутствующие закрывающие скобки

LLM зацикливается, добавляет кучу переносов/пробелов или обрывает ответ:

```json
{
  "key": "value"
  
  
.....
```

Или:

```json
{
  "user": {
    "name": "John",
    "items": [1, 2, 3
  
```

### 3. Мусор в конце JSON

```json
{"key": "value"}
\n\n\n.....
```

## Решение

Добавлена автоматическая предобработка JSON с **четырьмя типами исправлений**:

1. **Извлечение** - посимвольное извлечение валидного JSON из строки с мусором
2. **Запятые** - добавление отсутствующих запятых между полями
3. **Скобки** - добавление отсутствующих закрывающих скобок `}` и `]`
4. **Мусор** - удаление мусора в конце JSON (переносы, пробелы, точки)

### Как это работает

**Алгоритм исправления:**

0. **Извлечение JSON** - посимвольно идём по строке, считаем баланс скобок:
   - `{"key": "value"} extra text` → `{"key": "value"}`
   - `{"first": 1} {"second": 2}` → `{"first": 1}` (первый валидный)
   - `{"key": "value"}\n\n\ngarbage` → `{"key": "value"}`
   - Если баланс не 0 в конце - добавляем недостающие скобки

1. **Исправление запятых** - ищем паттерны где после значения отсутствует запятая:
   - `"value"\n"key"` → `"value",\n"key"`
   - `}\n"key"` → `},\n"key"`  
   - `]\n"key"` → `],\n"key"`
   - `123\n"key"` → `123,\n"key"`
   - `true/false/null\n"key"` → с запятой
   - `}\n{` → `},\n{` (объекты в массивах)

2. **Исправление скобок** - подсчитываем баланс и добавляем недостающие:
   - `{"key": "value"` → `{"key": "value"}`
   - `{"arr": [1, 2, 3` → `{"arr": [1, 2, 3]}`
   - `{"obj": {"inner": "value"}` → `{"obj": {"inner": "value"}}`
   - Учитывает строки и escape-символы (не считает скобки внутри строк)

3. **Удаление мусора** - обрезка всего после последней закрывающей скобки:
   - `{"key": "value"}\n\n\n\n.....` → `{"key": "value"}`
   - `{"key": "value"}      ` → `{"key": "value"}`

4. Применяем исправления через regex

5. Пробуем распарсить исправленный JSON

6. Логируем результат (успех/неудача)

### Примеры

**До исправления:**
```python
broken = '{"name": "test"\n"value": 42'
json.loads(broken)  # JSONDecodeError
```

**После исправления:**
```python
from core.infrastructure.providers.llm.json_parser import (
    _extract_and_fix_json,
    _fix_missing_commas,
    _fix_missing_closing_brackets,
    _fix_json_trailing_garbage
)

# Пример 1: JSON с мусором внутри
dirty = '{"name": "test"} extra garbage text'
extracted = _extract_and_fix_json(dirty)  # '{"name": "test"}'
data = json.loads(extracted)  # ✅ {"name": "test"}

# Пример 2: Несколько JSON объектов
dirty = '{"first": 1} {"second": 2}'
extracted = _extract_and_fix_json(dirty)  # '{"first": 1}'

# Пример 3: Реальный кейс с зацикливанием
dirty = '''{"thought": "Let me analyze", "confidence": 0.85}

I think this is correct. Let me add more explanation...
The analysis shows that...
random text here
'''
extracted = _extract_and_fix_json(dirty)
# Результат: '{"thought": "Let me analyze", "confidence": 0.85}'
```

**Примеры исправления:**

```python
# Пример 1: Отсутствующие запятые
broken = '{"a": 1\n"b": 2}'
fixed = _fix_missing_commas(broken)  # '{"a": 1,\n"b": 2}'

# Пример 2: Отсутствующие скобки
broken = '{"user": {"name": "John"'
fixed = _fix_missing_closing_brackets(broken)  # '{"user": {"name": "John"}}'

# Пример 3: Мусор в конце
broken = '{"key": "value"}\n\n\n.....'
fixed = _fix_json_trailing_garbage(broken)  # '{"key": "value"}'

# Пример 4: Комбинированный случай
broken = '{"name": "test"\n"items": [1, 2, 3\n\n\n'
fixed = _fix_missing_commas(broken)
fixed = _fix_missing_closing_brackets(fixed)
fixed = _fix_json_trailing_garbage(fixed)
# Результат: '{"name": "test",\n"items": [1, 2, 3]}'
```
