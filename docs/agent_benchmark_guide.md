# 📊 Бенчмарк для агента — Полное руководство

## ❓ Вопрос: Где ответ агента в бенчмарке?

**Ответ:** В бенчмарке **НЕТ ответа агента**. Бенчмарк содержит:
- ✅ Входные данные (natural language вопрос)
- ✅ Ожидаемые результаты (книги из БД)
- ✅ Правила валидации (какие keywords должны быть)
- ❌ **НЕ содержит** ответ агента

**Ответ агента генерируется когда мы ЗАПУСКАЕМ агента на бенчмарке.**

---

## 🔄 Полный цикл тестирования

```
┌─────────────────────────────────────────────────────────────────┐
│  1. БЕНЧМАРК (готовый файл)                                     │
│  {                                                              │
│    "input": "Какие книги написал Пушкин?",                     │
│    "expected_output": {"books": [...], "count": 5},            │
│    "validation": {"must_contain_keywords": [...]}              │
│  }                                                              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  2. ЗАПУСК АГЕНТА                                               │
│  agent.run("Какие книги написал Пушкин?")                      │
│                                                                  │
│  Агент:                                                          │
│  1. Генерирует SQL → SELECT * FROM books WHERE...              │
│  2. Выполняет SQL → получает 5 книг из БД                      │
│  3. Генерирует ответ → "Пушкин написал: Евгений Онегин..."     │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  3. СРАВНЕНИЕ С БЕНЧМАРКОМ                                      │
│                                                                  │
│  Ответ агента vs Бенчмарк:                                     │
│  ├─ SQL валидный? → ✅/❌                                       │
│  ├─ Книги совпадают? → ✅/❌                                    │
│  ├─ Keywords в ответе? → ✅/❌                                  │
│  ├─ На русском языке? → ✅/❌                                   │
│  └─ Нет галлюцинаций? → ✅/❌                                   │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  4. РЕЗУЛЬТАТЫ                                                  │
│  {                                                              │
│    "sql_generation": {"success_rate": 0.85},                   │
│    "final_answer": {"success_rate": 0.90}                      │
│  }                                                              │
└─────────────────────────────────────────────────────────────────┘
```

## 📊 Метрики

### SQL Generation

| Метрика | Что измеряет | Как считается |
|---------|--------------|---------------|
| `sql_valid` | Валидность SQL | Парсинг SQL |
| `has_where` | Наличие WHERE | `WHERE in SQL` |
| `has_join` | Наличие JOIN | `JOIN in SQL` |
| `correct_tables` | Правильные таблицы | Парсинг SQL |

### Final Answer

| Метрика | Что измеряет | Как считается |
|---------|--------------|---------------|
| `has_keywords` | Книги в ответе | Keywords in answer |
| `is_russian` | Язык ответа | Кириллица в тексте |
| `no_hallucination` | Нет выдумок | Сравнение с БД |
| `min_length` | Длина ответа | `len(answer) >= 20` |

## 🔍 Пример валидации

### Входные данные (из бенчмарка)

```json
{
  "input": "Какие книги написал Пушкин?",
  "validation": {
    "must_contain_keywords": ["Евгений Онегин", "Капитанская дочка"],
    "must_be_in_russian": true,
    "must_not_hallucinate": true
  }
}
```

### Ответ агента (генерируется при запуске)

```json
{
  "sql": "SELECT * FROM books WHERE author = 'Пушкин'",
  "sql_valid": true,
  "final_answer": "Пушкин написал: Евгений Онегин, Капитанская дочка, Руслан и Людмила"
}
```

### Валидация

```python
# Проверка keywords
required = ["Евгений Онегин", "Капитанская дочка"]
answer = "Пушкин написал: Евгений Онегин, Капитанская дочка, Руслан и Людмила"

missing = [kw for kw in required if kw not in answer]
# missing = [] → ✅ PASS

# Проверка языка
has_cyrillic = any('\u0400' <= c <= '\u04FF' for c in answer)
# has_cyrillic = True → ✅ PASS

# Итог
validation = {
  "passed": True,
  "errors": []
}
```

## ⚠️ Важно

### Что НЕ делает бенчмарк

| Не делает | Почему |
|-----------|--------|
| Не содержит ответ агента | Ответ генерируется при запуске |
| Не запускает агента | Это делает `run_agent_benchmark.py` |
| Не оценивает качество сам | Нужен скрипт валидации |

### Что НУЖНО сделать вам

1. **Реализовать `run_agent_on_input()`** — вызов реального агента
2. **Реализовать валидацию SQL** — парсинг SQL запроса
3. **Реализовать проверку на галлюцинации** — сравнение с БД
