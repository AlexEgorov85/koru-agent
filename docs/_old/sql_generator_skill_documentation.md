# Документация: Навык генерации SQL запросов

## Обзор

Навык генерации SQL запросов (`SQLGeneratorSkill`) предоставляет возможность автоматической генерации SQL запросов на основе описания таблиц и требований. Навык включает в себя проверку синтаксиса, построение плана выполнения запроса, автоматическое исправление ошибок и оптимизацию запросов.

## Функциональные возможности

### 1. Генерация SQL запросов
- Поддержка основных типов SQL запросов: SELECT, INSERT, UPDATE, DELETE
- Генерация на основе описания требований пользователя
- Использование информации о структуре таблиц

### 2. Проверка синтаксиса
- Проверка синтаксиса сгенерированных запросов
- Обнаружение потенциальных проблем с безопасностью
- Возврат детализированных ошибок и предупреждений

### 3. Автоматическое исправление ошибок
- Исправление типичных синтаксических ошибок
- Корректировка опечаток в ключевых словах
- Восстановление после частичных ошибок

### 4. Построение плана выполнения
- Получение плана выполнения для безопасных запросов
- Анализ эффективности запроса
- Возврат информации о планировании выполнения

### 5. Оптимизация запросов
- Базовая оптимизация структуры запроса
- Анализ на основе плана выполнения
- Повышение эффективности запроса

### 6. Поддержка различных диалектов SQL
- PostgreSQL
- MySQL
- SQLite
- Стандартный SQL

## Архитектура

### Классы и модели

#### `SQLGenerationParams`
Параметры для генерации SQL запроса:

- `description`: Описание требуемого SQL запроса
- `tables`: Список информации о таблицах
- `sql_type`: Тип SQL запроса (SELECT, INSERT, UPDATE, DELETE)
- `format`: Формат SQL (диалект)
- `validate_syntax`: Флаг проверки синтаксиса
- `optimize_query`: Флаг оптимизации запроса
- `explain_plan`: Флаг построения плана выполнения
- `auto_correct`: Флаг автоматического исправления ошибок

#### `TableInfo`
Информация о таблице:

- `name`: Имя таблицы
- `schema_name`: Имя схемы (по умолчанию "public")
- `description`: Описание таблицы
- `columns`: Список описаний колонок

#### `SQLGenerationResult`
Результат генерации SQL:

- `generated_sql`: Сгенерированный SQL запрос
- `validation_result`: Результат валидации
- `optimization_result`: Результат оптимизации (если применимо)
- `execution_plan`: План выполнения запроса
- `corrected`: Флаг корректировки запроса
- `correction_reason`: Причина корректировки

#### `SQLValidationResult`
Результат проверки синтаксиса:

- `is_valid`: Флаг валидности запроса
- `errors`: Список ошибок
- `warnings`: Список предупреждений
- `suggestions`: Рекомендации по улучшению

#### `SQLOptimizationResult`
Результат оптимизации:

- `original_query`: Оригинальный запрос
- `optimized_query`: Оптимизированный запрос
- `performance_improvement`: Улучшение производительности
- `execution_plan`: План выполнения запроса

## Использование

### Пример использования

```python
from core.skills.sql_generator.skill import SQLGeneratorSkill
from core.skills.sql_generator.schema import SQLGenerationParams, TableInfo

# Создание экземпляра навыка
sql_generator = SQLGeneratorSkill("sql_generator", system_context)

# Подготовка параметров
params = SQLGenerationParams(
    description="Выбрать ID, имя и email всех активных пользователей",
    tables=[
        TableInfo(
            name="users",
            schema_name="public",
            description="Таблица пользователей",
            columns=[
                {"name": "id", "type": "integer", "description": "ID пользователя"},
                {"name": "name", "type": "varchar", "description": "Имя пользователя"},
                {"name": "email", "type": "varchar", "description": "Email пользователя"},
                {"name": "active", "type": "boolean", "description": "Статус активности"}
            ]
        )
    ],
    sql_type="SELECT",
    format="standard",
    validate_syntax=True,
    optimize_query=True,
    explain_plan=True,
    auto_correct=True
)

# Выполнение генерации
capability = sql_generator.get_capabilities()[0]
result = await sql_generator.execute(capability, params, context)
```

### Возможности интеграции

Навык предназначен для использования другими навыками в их доменных областях. Он может быть вызван через систему capability следующим образом:

```python
# Вызов из другого навыка
sql_result = await self.system_context.execute_capability(
    capability_name="sql_generator.generate",
    parameters=params_dict
)
```

## Безопасность

Навык включает несколько уровней безопасности:

1. **Проверка паттернов**: Обнаружение потенциально опасных SQL паттернов
2. **Ограничение операций**: Безопасное выполнение EXPLAIN только для безопасных запросов
3. **Валидация входных данных**: Проверка структуры параметров через Pydantic

## Обработка ошибок

Навык реализует комплексную обработку ошибок:

1. **Синтаксические ошибки**: Автоматическое исправление
2. **Ошибки валидации**: Подробные сообщения об ошибках
3. **Ошибки выполнения**: Fallback к безопасным значениям
4. **Ошибки LLM**: Резервные механизмы генерации
