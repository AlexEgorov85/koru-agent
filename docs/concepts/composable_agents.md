# Компонуемые агенты

Компонуемые агенты (Composable Agents) - это центральный компонент Composable AI Agent Framework, позволяющий комбинировать различные паттерны мышления для решения сложных задач. В отличие от традиционных подходов, где LLM управляет системой напрямую, компонуемые агенты обеспечивают структурированный подход с жесткой валидацией решений.

## Определение

Компонуемый агент - это программная сущность, которая:
- Может выполнять атомарные действия
- Может выполнять компонуемые паттерны мышления
- Адаптируется к различным доменам задач
- Работает в различных стратегиях решения задач
- Поддерживает управление состоянием и восстановление после ошибок

## Архитектура компонуемого агента

Компонуемый агент состоит из следующих ключевых компонентов:

### 1. Runtime (Среда выполнения)
- Управляет жизненным циклом агента
- Обрабатывает состояние агента
- Координирует выполнение паттернов и действий

### 2. State Manager (Менеджер состояния)
- Отслеживает текущее состояние агента
- Регистрирует ошибки и прогресс
- Обеспечивает восстановление после сбоев

### 3. Pattern Executor (Исполнитель паттернов)
- Выполняет зарегистрированные паттерны мышления
- Управляет переходами между паттернами
- Обрабатывает результаты выполнения

## Интерфейс агента

Компонуемые агенты реализуют интерфейс, определенный в `domain/agents/base_agent.py` и `domain/agents/runtime_interface.py`:

```python
class AgentState(BaseModel):
    """
    Явное состояние агента.
    Не содержит логики — только данные.
    """

    step: int = 0
    error_count: int = 0
    no_progress_steps: int = 0
    finished: bool = False
    metrics: Dict[str, Any] = {}
    history: List[str] = []
    current_plan_step: Optional[str] = None

    def register_error(self):
        self.error_count += 1

    def register_progress(self, progressed: bool):
        if progressed:
            self.no_progress_steps = 0
        else:
            self.no_progress_steps += 1

    def complete(self):
        """Отмечает агента как завершившего выполнение."""
        self.finished = True
```

## Основные возможности

### 1. Выполнение атомарных действий
Компонуемые агенты могут выполнять атомарные действия, определенные в системе:

```python
# Пример выполнения атомарного действия
result = await agent.execute_atomic_action(
    action_name="file_reader",
    parameters={"path": "example.txt"}
)
```

### 2. Выполнение компонуемых паттернов
Агенты могут выполнять различные паттерны мышления:

```python
# Пример выполнения паттерна мышления
result = await agent.execute_composable_pattern(
    pattern_name="analysis_pattern",
    context={"data": input_data}
)
```

### 3. Адаптация к доменам
Агенты могут адаптироваться к различным доменам задач:

```python
# Пример адаптации к домену
await agent.adapt_to_domain(
    domain_type="code_analysis",
    capabilities=["code_reading", "pattern_recognition"]
)
```

## Варианты реализации

### 1. ComposableAgent (Полный функционал)
- Поддерживает все возможности фреймворка
- Включает продвинутые механизмы управления состоянием
- Поддерживает сложные сценарии компоновки паттернов

### 2. SimpleComposableAgent (Упрощенный)
- Упрощенная реализация для базовых задач
- Меньше возможностей по настройке
- Подходит для начального использования

## Состояние агента

Каждый агент имеет явное состояние, которое включает:

- **step**: Текущий шаг выполнения
- **error_count**: Количество ошибок
- **no_progress_steps**: Количество шагов без прогресса
- **finished**: Флаг завершения выполнения
- **metrics**: Метрики выполнения
- **history**: История действий
- **current_plan_step**: Текущий шаг плана

## Управление состоянием

Агенты включают встроенную систему управления состоянием:

```python
# Регистрация ошибки
agent.state.register_error()

# Регистрация прогресса
agent.state.register_progress(True)

# Пометка завершения
agent.state.complete()
```

## Восстановление после ошибок

Система включает механизм восстановления паттернов мышления после ошибок, реализованный в `PatternRecoveryManager`:

```python
class PatternRecoveryManager:
    """Менеджер восстановления паттернов мышления после ошибок."""
    
    async def create_pattern_checkpoint(
        self,
        agent: Any,  # AgentRuntime
        pattern_name: str
    ) -> str:
        """Создать чекпоинт состояния паттерна мышления."""
    
    async def restore_pattern_from_checkpoint(
        self,
        agent: Any,  # AgentRuntime
        checkpoint_id: str
    ) -> bool:
        """Восстановить паттерн мышления из чекпоинта."""
```

## Интеграция с паттернами мышления

Компонуемые агенты тесно интегрированы с системой паттернов мышления:

- Агент выбирает подходящий паттерн на основе текущего состояния
- Паттерн выполняется в контексте агента
- Результат паттерна влияет на дальнейшее поведение агента
- Состояние агента обновляется на основе результата паттерна

## Пример использования

```python
from application.factories.agent_factory import AgentFactory
from domain.value_objects.domain_type import DomainType

# Создание агента через фабрику
factory = AgentFactory()
agent = await factory.create_agent(
    agent_type="composable",
    domain=DomainType.CODE_ANALYSIS
)

# Адаптация агента к задаче
await agent.adapt_to_task(
    task_description="Анализировать код на Python для поиска уязвимостей"
)

# Выполнение атомарного действия
file_content = await agent.execute_atomic_action(
    action_name="file_reader",
    parameters={
        "path": "/path/to/python/file.py"
    }
)

# Выполнение паттерна мышления
analysis_result = await agent.execute_composable_pattern(
    pattern_name="security_analysis",
    context={
        "code": file_content,
        "task": "Поиск уязвимостей в Python коде"
    }
)

# Проверка завершения
if agent.state.finished:
    print("Агент завершил выполнение задачи")
```

## Преимущества компонуемых агентов

- **Гибкость**: Возможность комбинирования различных стратегий решения задач
- **Контролируемость**: Жесткая валидация решений, принятых LLM
- **Надежность**: Встроенные механизмы восстановления после ошибок
- **Масштабируемость**: Возможность адаптации к различным доменам задач
- **Прозрачность**: Явное состояние и история выполнения
- **Повторное использование**: Переиспользование паттернов в разных контекстах

## Интеграция с остальной системой

Компонуемые агенты интегрированы с:
- Системой паттернов мышления
- Системой управления промтами
- Системой атомарных действий
- Системой инструментов и навыков
- Системой событий и оркестрации
