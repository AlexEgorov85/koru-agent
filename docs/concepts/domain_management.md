# Управление доменами

Управление доменами (Domain Management) - это ключевой компонент Composable AI Agent Framework, позволяющий агентам адаптироваться к различным областям задач и контекстам. Система управления доменами обеспечивает гибкость и универсальность агентов, позволяя им эффективно работать в различных предметных областях.

## Определение

Домен - это сфера деятельности или область знаний, в которой агент выполняет задачи. Управление доменами включает:
- Классификацию задач по доменам
- Адаптацию агента к специфике домена
- Управление наборами возможностей (capabilities) для каждого домена
- Интеграцию с соответствующими промтами инструментами

## Архитектура управления доменами

Система управления доменами включает следующие компоненты:

### 1. Менеджер доменов
- Центральный компонент управления доменами
- Отвечает за регистрацию, идентификацию и переключение доменов
- Обеспечивает доступ к доменно-специфическим ресурсам

### 2. Регистр доменов
- Хранит информацию о доступных доменах
- Управляет метаданными доменов
- Обеспечивает сопоставление задач с подходящими доменами

### 3. Адаптеры доменов
- Обеспечивают интеграцию с доменно-специфическими компонентами
- Адаптируют поведение агента к требованиям домена
- Управляют доменно-специфическими промтами и инструментами

## Типы доменов

Система поддерживает различные типы доменов:

### 1. Код-анализ (Code Analysis)
- Назначение: Анализ исходного кода
- Возможности: Чтение кода, анализ AST, поиск паттернов, рефакторинг
- Примеры задач: Поиск уязвимостей, рефакторинг, генерация документации

### 2. Обработка данных (Data Processing)
- Назначение: Обработка и анализ данных
- Возможности: Чтение файлов, выполнение SQL-запросов, трансформация данных
- Примеры задач: ETL-процессы, анализ данных, валидация данных

### 3. Генерация контента (Content Generation)
- Назначение: Создание текстового или другого контента
- Возможности: Генерация текста, форматирование, валидация
- Примеры задач: Написание отчетов, генерация документации, создание текстов

### 4. Инфраструктура (Infrastructure)
- Назначение: Управление инфраструктурой
- Возможности: Выполнение команд, управление процессами, мониторинг
- Примеры задач: Деплой, мониторинг, диагностика

## Типы доменов (DomainType)

Система использует перечисление `DomainType` для определения доступных доменов:

```python
from enum import Enum

class DomainType(Enum):
    """Типы доменов"""
    CODE_ANALYSIS = "code_analysis"
    DATA_PROCESSING = "data_processing"
    CONTENT_GENERATION = "content_generation"
    INFRASTRUCTURE = "infrastructure"
    SECURITY = "security"
    TESTING = "testing"
    DOCUMENTATION = "documentation"
```

## Возможности домена (Capabilities)

Каждый домен определяет набор возможностей (capabilities), которые агент может использовать:

```python
# Пример возможностей для домена анализа кода
CODE_ANALYSIS_CAPABILITIES = [
    "file_reading",
    "ast_parsing",
    "pattern_matching",
    "code_generation",
    "refactoring",
    "security_analysis"
]

# Пример возможностей для домена обработки данных
DATA_PROCESSING_CAPABILITIES = [
    "file_operations",
    "sql_execution",
    "data_validation",
    "data_transformation",
    "report_generation"
]
```

## Адаптация агента к домену

Процесс адаптации агента к домену включает:

### 1. Определение домена
- Анализ задачи для определения наиболее подходящего домена
- Выбор домена на основе ключевых слов и контекста задачи

### 2. Настройка возможностей
- Активация доменно-специфических возможностей
- Настройка доступных атомарных действий

### 3. Загрузка промтов
- Загрузка промтов, соответствующих домену
- Настройка параметров промтов для конкретной задачи

### 4. Конфигурация паттернов мышления
- Активация подходящих паттернов мышления
- Настройка параметров паттернов

## Пример использования

```python
from application.domain_management.domain_manager import DomainManager
from domain.value_objects.domain_type import DomainType

# Создание менеджера доменов
domain_manager = DomainManager()

# Определение домена для задачи
task_description = "Анализировать Python код для поиска уязвимостей"
domain = await domain_manager.identify_domain(task_description)

print(f"Определенный домен: {domain}")

# Адаптация агента к домену
agent_state = AgentState()  # Текущее состояние агента
adaptation_success = await domain_manager.adapt_agent_to_domain(
    agent_state, 
    domain
)

if adaptation_success:
    # Получение возможностей домена
    capabilities = domain_manager.get_capabilities_for_domain(domain)
    print(f"Доступные возможности: {capabilities}")
    
    # Получение промтов домена
    prompts = domain_manager.get_prompts_for_domain(domain)
    print(f"Доступные промты: {prompts}")
```

## Интеграция с агентами

Управление доменами тесно интегрировано с агентами:

```python
# Адаптация агента к домену
await agent.adapt_to_domain(
    domain_type=DomainType.CODE_ANALYSIS,
    capabilities=["code_reading", "pattern_recognition", "security_analysis"]
)

# Использование доменно-специфических возможностей
result = await agent.execute_atomic_action(
    action_name="ast_parser",
    parameters={"code": source_code}
)
```

## Валидация доменов

Система включает механизмы валидации доменов:

- **Проверка соответствия задачи домену**: Убедиться, что задача действительно относится к определенному домену
- **Проверка доступности возможностей**: Убедиться, что все необходимые возможности доступны в домене
- **Проверка совместимости промтов**: Убедиться, что промты совместимы с задачей

## Преимущества системы управления доменами

- **Гибкость**: Возможность адаптации агентов к различным областям задач
- **Эффективность**: Использование доменно-специфических стратегий решения задач
- **Масштабируемость**: Легкое добавление новых доменов и возможностей
- **Модульность**: Четкое разделение функциональности по доменам
- **Прозрачность**: Ясное понимание, в каком домене работает агент
- **Контролируемость**: Возможность ограничения функциональности по доменам

## Расширение системы доменов

Для добавления нового домена необходимо:

1. Определить новый тип домена в `DomainType`
2. Зарегистрировать домен в менеджере доменов
3. Определить набор возможностей для домена
4. Создать доменно-специфические промты
5. Реализовать доменно-специфические паттерны мышления (при необходимости)

## Интеграция с другими компонентами

Управление доменами интегрировано с:
- **Системой паттернов мышления**: Активация подходящих паттернов
- **Системой управления промтами**: Загрузка доменно-специфических промтов
- **Атомарными действиями**: Активация подходящих действий
- **Системой инструментов**: Доступ к доменно-специфическим инструментам
- **Управлением состоянием**: Адаптация состояния к домену
