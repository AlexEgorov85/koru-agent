# 🤖 Автоматическая оптимизация промптов — Полное руководство

## 📋 Обзор

Теперь система поддерживает **два режима работы**:

| Режим | Что делает | Когда использовать |
|-------|------------|-------------------|
| **Анализ** | Находит проблемы, даёт рекомендации | Для понимания что улучшать |
| **Автоматизация** | Сам исправляет, тестирует, сохраняет | Для production оптимизации |

---

## 🔄 Полный цикл автоматизации

```
┌─────────────────────────────────────────────────────────────────┐
│  1. АНАЛИЗ ЛОГОВ                                                │
│     - Сбор execution traces                                     │
│     - Поиск паттернов (repeated_retry, timeout, etc.)          │
│     - Поиск корневых причин                                     │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  2. LLM ГЕНЕРАЦИЯ УЛУЧШЕНИЙ                                     │
│     - Взять проблемный промпт                                   │
│     - Сгенерировать улучшенную версию через LLM                │
│     - На основе root causes и примеров                          │
│     - Детерминированно (temperature=0, seed=42)                │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  3. A/B ТЕСТИРОВАНИЕ                                            │
│     - Запустить старую и новую версии на одних данных          │
│     - Сравнить метрики (success_rate, time, tokens)            │
│     - Safety check (не стало ли хуже)                           │
│     - Статистическая значимость                                 │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  4. ПРОДВИЖЕНИЕ ВЕРСИИ                                          │
│     - Если новая лучше → сохранить                              │
│     - Записать в data/prompts/...                              │
│     - Записать метрики улучшения                                │
│     - Создать YAML файл с metadata                              │
└─────────────────────────────────────────────────────────────────┘
```

## 🚀 Использование

### Режим 1: Анализ (существующий)

```bash
# Запустить анализ
py -m scripts.cli.analyze_and_optimize

# Посмотреть отчёт
cat data/optimization_report.json
```

**Результат:** Рекомендации что исправить

---

### Режим 2: Автоматизация (новый)

```bash
# Автоматическая оптимизация
py scripts/cli/auto_optimize.py \
    --capability final_answer.generate \
    --focus error_handling

# Тестовый режим (без сохранения)
py scripts/cli/auto_optimize.py \
    --capability final_answer.generate \
    --focus error_handling \
    --dry-run
```

**Результат:** Новая версия промпта сохранена в `data/prompts/...`

## 📈 Метрики

### A/B тест сравнивает:

| Метрика | Что измеряет | Лучше когда |
|---------|--------------|-------------|
| `success_rate` | Доля успешных выполнений | ↑ Больше |
| `execution_time` | Время выполнения | ↓ Меньше |
| `tokens` | Использовано токенов | ↓ Меньше |

### Критерии победы:

```python
# Новая версия побеждает если:
success_rate улучшился на ≥5%  # min_improvement=0.05
ИЛИ
execution_time уменьшился на ≥5%

# Новая версия проигрывает если:
success_rate ухудшился на ≥2%  # max_regression=0.02
```

## 🔧 Конфигурация

### Настройки A/B теста

```python
@dataclass
class ABTestConfig:
    min_improvement: float = 0.05    # 5% минимальное улучшение
    max_regression: float = 0.02     # 2% максимально допустимое ухудшение
    num_runs: int = 3                # Количество запусков
    timeout_seconds: int = 60        # Таймаут на запуск
```

### Настройки LLM генерации

```python
# В PromptImprover
response = await self.llm_provider.generate(
    prompt=generation_prompt,
    temperature=0.0,    # Детерминированно
    max_tokens=4000,
    seed=42             # Фиксированный seed
)
```

## ⚠️ Важные замечания

### Безопасность

1. **Dry run режим** — всегда тестируйте сначала с `--dry-run`
2. **A/B тест** — новая версия не сохранится если не прошла тест
3. **Safety check** — ухудшение >2% автоматически отменяет продвижение

### Детерминированность

1. **temperature=0** — LLM генерирует одинаково при тех же входах
2. **seed=42** — фиксированный seed для воспроизводимости
3. **Хеш в версии** — `v1.0.1_a1b2c3d4` уникален для содержимого

### Ограничения

1. **Нужны логи** — без данных в `data/logs/sessions/` не работает
2. **Mock executor** — сейчас A/B тест использует mock, нужен реальный
3. **Один промпт** — оптимизирует один capability за запуск
