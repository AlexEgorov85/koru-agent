## 🏗️ Архитектурные решения (ЭТАП 0)

### Стек технологий

```
┌─────────────────────────────────────────────────────────┐
│                    VECTOR SEARCH STACK                  │
├─────────────────────────────────────────────────────────┤
│  FAISS (основной индекс)                                │
│  +                                                      │
│  JSON (персистентность метаданных и chunking)           │
│  +                                                      │
│  SentenceTransformers (локальные эмбеддинги)           │
└─────────────────────────────────────────────────────────┘
```

### Компоненты

| Компонент | Расположение | Описание |
|-----------|--------------|----------|
| `FAISSProvider` | Infrastructure | Работа с индексом FAISS |
| `EmbeddingProvider` | Infrastructure | Генерация эмбеддингов (SentenceTransformers) |
| `VectorSearchService` | Application | Бизнес-логика поиска |
| `DocumentManager` | Application | Добавление/удаление документов |
| `VectorTool` | Application | Инструмент для агента |
| `ChunkingService` | Application | Разбиение документов на чанки |

### Архитектура

```
┌──────────────────────────────────────────────────────────────┐
│                         АГЕНТ                                │
└──────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌──────────────────────────────────────────────────────────────┐
│                    VectorTool                                │
└──────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌──────────────────────────────────────────────────────────────┐
│                 VectorSearchService                          │
│  ┌────────────────┐  ┌────────────────┐  ┌────────────────┐ │
│  │   Поиск        │  │   Фильтрация   │  │   Метрики      │ │
│  └────────────────┘  └────────────────┘  └────────────────┘ │
└──────────────────────────────────────────────────────────────┘
                            │
            ┌───────────────┼───────────────┐
            ▼               ▼               ▼
┌──────────────────┐ ┌──────────────────┐ ┌──────────────────┐
│  FAISSProvider   │ │  Embedding       │ │  Chunking        │
│  (индекс)        │ │  Provider        │ │  Service         │
│                  │ │  (эмбеддинги)    │ │  (чанки)         │
└──────────────────┘ └──────────────────┘ └──────────────────┘
        │
        ▼
┌──────────────────┐
│  FAISS Index     │
│  + JSON Metadata │
└──────────────────┘
```

### Решения

| Решение | Обоснование |
|---------|-------------|
| **FAISS** вместо Qdrant/Pinecone | Локальное хранение, нет внешних зависимостей, полный контроль |
| **JSON** для метаданных | Простота, читаемость, не требует дополнительной БД |
| **SentenceTransformers** для эмбеддингов | Локальная модель, нет зависимости от API, бесплатно |
| **Chunking** с перекрытием | Улучшает поиск по большим документам |
| **Event-driven** синхронизация | Соответствует архитектуре проекта |

## 🎯 Критерии приёмки (Acceptance Criteria)

### Функциональные:
```
□ Поиск по тексту работает (query → embedding → search)
□ Поиск по вектору работает (vector → search)
□ Фильтрация по метаданным работает
□ Возврат документов с полным содержимым
□ Обработка ошибок (таймауты, недоступность БД)
```

### Нефункциональные:
```
□ Время поиска p95 < 500ms
□ Точность поиска recall@10 > 0.9
□ Поддержка ≥ 100,000 документов
□ Масштабируемость (горизонтальное масштабирование)
```

### Архитектурные:
```
□ Провайдер не привязан к конкретному вендору
□ Изоляция кэшей между агентами
□ Интеграция с EventBus
□ Интеграция с MetricsCollector
□ Соответствие текущей архитектуре проекта
```

### Тестовые:
```
□ Покрытие тестами ≥ 85%
□ Все тесты проходят в CI/CD
□ Mock провайдер для тестов без БД
□ Performance тесты в pipeline
```

## 🔄 Процесс разработки (TDD цикл)

```
┌─────────────────────────────────────────────────────────────┐
│                    TDD ЦИКЛ ДЛЯ КАЖДОЙ ФУНКЦИИ              │
└─────────────────────────────────────────────────────────────┘

1. НАПИСАТЬ ТЕСТ
   └→ Тест описывает ожидаемое поведение
   └→ Тест падает (красный)
   │
2. НАПИСАТЬ МИНИМАЛЬНУЮ РЕАЛИЗАЦИЮ
   └→ Только чтобы тест прошёл
   └→ Не заботиться о качестве кода
   │
3. ЗАПУСТИТЬ ТЕСТ
   └→ Тест проходит (зелёный)
   └→ Если нет → вернуться к шагу 2
   │
4. РЕФАКТОРИНГ
   └→ Улучшить код
   └→ Убедиться что тесты всё ещё проходят
   │
5. ПОВТОРИТЬ
   └→ Следующая функция/метод
```
