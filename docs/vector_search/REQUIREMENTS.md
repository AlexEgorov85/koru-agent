## 📋 Обзор

Этот документ описывает функциональные и нефункциональные требования к системе векторного поиска в Agent_v5.

---

## 🎯 Функциональные требования

### FR-001: Поиск по тексту

**Описание:** Система должна поддерживать поиск документов по текстовому запросу.

**Входные данные:**
```python
{
    "query": "текст запроса",
    "top_k": 10,                    # количество результатов
    "min_score": 0.5,               # минимальный порог схожести
    "filters": {                    # опциональные фильтры
        "category": ["knowledge"],
        "tags": ["agent"],
        "date_from": "2026-01-01",
        "date_to": "2026-12-31"
    }
}
```

**Выходные данные:**
```python
{
    "results": [
        {
            "id": "doc_123",
            "score": 0.92,
            "content": "содержимое документа или чанка",
            "metadata": {
                "category": "knowledge",
                "created_at": "2026-02-19T10:00:00Z",
                "tags": ["agent", "search"]
            }
        }
    ],
    "total_found": 15,
    "query_time_ms": 45
}
```

**Критерии приёмки:**
- [ ] Поиск возвращает релевантные документы
- [ ] Результаты отсортированы по убыванию score
- [ ] Фильтры применяются корректно
- [ ] Время поиска < 1000ms (p95)

---

### FR-002: Поиск по вектору

**Описание:** Система должна поддерживать поиск документов по готовому вектору.

**Входные данные:**
```python
{
    "vector": [0.1, 0.2, ..., 0.5],  # вектор размерности 384
    "top_k": 10,
    "min_score": 0.5,
    "filters": {...}
}
```

**Выходные данные:** Аналогично FR-001

**Критерии приёмки:**
- [ ] Поиск по вектору возвращает те же результаты, что поиск по тексту с тем же вектором
- [ ] Поддерживаются все фильтры

---

### FR-003: Добавление документа

**Описание:** Система должна поддерживать добавление новых документов в индекс.

**Входные данные:**
```python
{
    "content": "текст документа",
    "metadata": {
        "category": "knowledge",
        "tags": ["agent"],
        "author": "user_123"
    },
    "chunk_size": 500,        # опционально
    "chunk_overlap": 50       # опционально
}
```

**Выходные данные:**
```python
{
    "document_id": "doc_123",
    "chunk_ids": ["chunk_1", "chunk_2", "chunk_3"],
    "status": "indexed"
}
```

**Критерии приёмки:**
- [ ] Документ разбивается на чанки
- [ ] Для каждого чанка генерируется эмбеддинг
- [ ] Чанки добавляются в индекс
- [ ] Метаданные сохраняются в JSON
- [ ] Генерируется событие VectorDocumentAdded

---

### FR-004: Обновление документа

**Описание:** Система должна поддерживать обновление существующих документов.

**Входные данные:** Аналогично FR-003 + document_id

**Выходные данные:**
```python
{
    "document_id": "doc_123",
    "chunk_ids": ["chunk_1", "chunk_2", "chunk_3"],
    "status": "updated"
}
```

**Критерии приёмки:**
- [ ] Старые чанки удаляются из индекса
- [ ] Новые чанки добавляются
- [ ] Метаданные обновляются
- [ ] Генерируется событие VectorDocumentUpdated

---

### FR-005: Удаление документа

**Описание:** Система должна поддерживать удаление документов.

**Входные данные:**
```python
{
    "document_id": "doc_123"
}
```

**Выходные данные:**
```python
{
    "document_id": "doc_123",
    "deleted_chunks": 3,
    "status": "deleted"
}
```

**Критерии приёмки:**
- [ ] Чанки удаляются из индекса
- [ ] Метаданные удаляются из JSON
- [ ] Генерируется событие VectorDocumentDeleted

---

### FR-006: Фильтрация результатов

**Описание:** Система должна поддерживать фильтрацию результатов поиска.

**Поддерживаемые фильтры:**

| Фильтр | Тип | Описание |
|--------|-----|----------|
| `category` | list[str] | Фильтр по категориям |
| `tags` | list[str] | Фильтр по тегам |
| `date_from` | str | Документы не раньше даты |
| `date_to` | str | Документы не позже даты |
| `author` | str | Фильтр по автору |

**Критерии приёмки:**
- [ ] Фильтры комбинируются (AND)
- [ ] Пустые фильтры игнорируются
- [ ] Несуществующие категории/теги возвращают пустой результат

---

### FR-007: Пагинация результатов

**Описание:** Система должна поддерживать пагинацию результатов.

**Входные данные:**
```python
{
    "query": "текст",
    "top_k": 10,
    "offset": 0  # смещение
}
```

**Критерии приёмки:**
- [ ] Корректная пагинация результатов
- [ ] Поддержка больших offset

---

## 🔧 Конфигурация

### Параметры конфигурации

```yaml
vector_search:
  enabled: true
  
  # FAISS индекс
  index:
    type: "IVF"              # IVF, HNSW, Flat
    nlist: 100               # количество кластеров для IVF
    nprobe: 10               # количество кластеров для поиска
    
  # Эмбеддинги
  embedding:
    model: "all-MiniLM-L6-v2"
    dimension: 384
    device: "cpu"            # cpu, cuda
    
  # Chunking
  chunking:
    enabled: true
    chunk_size: 500          # символов
    chunk_overlap: 50        # символов
    
  # Поиск
  search:
    default_top_k: 10
    max_top_k: 100
    default_min_score: 0.5
    recall_target: 0.90
    
  # Хранение
  storage:
    index_path: "./data/vector/index.faiss"
    metadata_path: "./data/vector/metadata.json"
    
  # Производительность
  performance:
    batch_size: 32           # размер батча для эмбеддингов
    max_workers: 4           # количество воркеров
```
