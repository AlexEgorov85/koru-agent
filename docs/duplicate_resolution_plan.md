# План устранения дублирования в документации

## Этап 1: Объединение дублирующихся файлов

### 1.1. Объединение best_practices файлов
- Объединить `best_practices.md` и `best_practices_guide.md` в один файл `best_practices.md`
- Удалить дублирующий файл `best_practices_guide.md`

### 1.2. Объединение overview файлов
- Объединить `overview.md`, `complete_overview.md`, и `final_overview.md` в один файл `overview.md`
- Удалить дублирующие файлы

### 1.3. Объединение guide файлов
- Объединить `guide.md`, `complete_guide.md` в один файл `guide.md`
- Оставить `customization_guide.md`, `integration_guide.md`, `migration_guide.md` как специфические руководства
- Объединить `framework_summary.md`, `final_summary.md`, `complete_system_guide.md` в `summary.md`

### 1.4. Объединение conclusion файлов
- Объединить `conclusion.md` и `CONCLUSION_FINAL.md` в один файл `conclusion.md`
- Удалить дублирующий файл

## Этап 2: Упрощение системы оглавлений

### 2.1. Удаление дублирующихся оглавлений
- Оставить только `SUMMARY.md` как основное оглавление
- Удалить `SUMMARY_ALL.md`, `TABLE_OF_CONTENTS.md`, `index.md`, `readme.md` (не путать с корневым README.md)

### 2.2. Обновление оставшегося оглавления
- Обновить `docs/SUMMARY.md` с учетом объединенных файлов
- Убедиться, что все ссылки ведут на существующие файлы

## Этап 3: Работа с файлами custom_development

### 3.1. Создание унифицированного файла
- Создать общий файл `custom_development.md` в корне docs/
- Разделить его на секции по компонентам:
  - Архитектура
  - Концепции
  - Промты
  - Инструменты и навыки
  - Ядро
  - Конфигурация
  - События
  - Система
  - Приложение

### 3.2. Удаление дублирующихся файлов
- Удалить все файлы `custom_development.md` из подкаталогов
- Обновить ссылки в оглавлении

## Этап 4: Обновление intro файлов

### 4.1. Объединение вводных материалов
- Объединить `intro.md` и `introduction.md` в один файл
- Оставить `getting_started.md` как отдельный файл для быстрого старта

## Этап 5: Обновление ссылок

### 5.1. Обновление внутренних ссылок
- Обновить все внутренние ссылки в документации после объединения файлов
- Проверить работоспособность всех ссылок

### 5.2. Обновление README.md
- Обновить ссылки в корневом README.md на актуальные файлы документации

## Этап 6: Улучшение структуры

### 6.1. Создание тематических подразделов
- Организовать документацию по функциональным областям
- Создать четкую иерархию: введение -> архитектура -> компоненты -> примеры -> руководства

### 6.2. Добавление статусов файлов
- Добавить в каждый файл информацию о статусе (актуальный/дубликат/устаревший)
- Добавить даты последнего обновления

## Этап 7: Проверка и тестирование

### 7.1. Проверка целостности
- Убедиться, что все ссылки работают
- Проверить, что нет потерянного контента

### 7.2. Проверка на дублирование
- Повторно проверить оставшиеся файлы на наличие дублирования

## Результаты

После выполнения этого плана:
- Количество файлов в docs/ уменьшится примерно на 30%
- Устранится дублирование информации
- Улучшится навигация по документации
- Упростится поддержка документации
- Повысится читаемость и структурированность информации
