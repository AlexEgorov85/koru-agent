# Введение в Koru AI Agent Framework

Koru AI Agent Framework - это гибкая и расширяемая система для создания, управления и выполнения AI-агентов, способных решать сложные задачи через компонуемые паттерны мышления. Фреймворк реализует архитектуру, в которой LLM не управляет системой напрямую, а принимает решения, которые строго валидируются и исполняются кодом.

## Основная концепция

Koru AI Agent Framework отличается от традиционных подходов к использованию LLM тем, что:

- **LLM не управляет системой напрямую**: LLM принимает решения, которые затем валидируются и исполняются кодом
- **Компонуемость**: Возможность комбинировать различные паттерны мышления для решения сложных задач
- **Контролируемость**: Все действия строго контролируются и ограничены системой
- **Безопасность**: Встроенные механизмы проверки безопасности и ограничения доступа
- **Расширяемость**: Легкое создание новых компонентов и адаптация под специфические задачи

## Архитектура фреймворка

Фреймворк реализует принципы чистой архитектуры (Clean Architecture):

```
┌─────────────────┐
│  Infrastructure │ ← Зависит от Application
│        ▲        │
│        │        │
├─────────────────┤
│   Application   │ ← Зависит от Domain
│        ▲        │
│        │        │
├─────────────────┤
│     Domain      │ ← Ядро системы, не зависит ни от чего
│                 │
└─────────────────┘
```

### Слой домена (Domain Layer)

Слой домена содержит:
- Бизнес-логику и правила
- Модели предметной области
- Абстракции для внешних зависимостей
- Паттерны мышления

### Слой приложений (Application Layer)

Слой приложений:
- Координирует работу компонентов домена
- Реализует сценарии использования
- Не содержит бизнес-логики
- Включает фабрики, сервисы и оркестрацию

### Слой инфраструктуры (Infrastructure Layer)

Слой инфраструктуры:
- Реализует внешние зависимости (базы данных, API, файловая система)
- Содержит адаптеры для внешних систем
- Не влияет на бизнес-логику
- Включает инструменты и шлюзы

## Основные компоненты

### 1. Компонуемые агенты

Компонуемые агенты - это основные исполнители логики в системе:

```python
from application.factories.agent_factory import AgentFactory
from domain.value_objects.domain_type import DomainType

# Создание агента
agent = await AgentFactory().create_agent(
    agent_type="composable",
    domain=DomainType.CODE_ANALYSIS
)

# Выполнение задачи
result = await agent.execute_task(
    task_description="Проанализируй этот Python код на наличие уязвимостей безопасности",
    context={
        "code": """
def login(username, password):
    query = f"SELECT * FROM users WHERE username='{username}' AND password='{password}'"
    return execute_query(query)
""",
        "language": "python"
    }
)
```

### 2. Паттерны мышления

Паттерны мышления определяют стратегии решения задач:

```python
from domain.abstractions.thinking_pattern import IThinkingPattern

class SecurityAnalysisPattern(IThinkingPattern):
    @property
    def name(self) -> str:
        return "security_analysis_pattern"
    
    async def execute(self, state, context, available_capabilities):
        # Логика выполнения анализа безопасности
        pass
```

### 3. Атомарные действия

Атомарные действия - минимальные неделимые операции:

```python
from domain.abstractions.atomic_action import IAtomicAction

class FileReaderAction(IAtomicAction):
    @property
    def name(self) -> str:
        return "file_reader"
    
    async def execute(self, parameters):
        # Логика чтения файла
        pass
```

### 4. Система промтов

Система промтов обеспечивает гибкое управление и версионирование:

```python
# Структура хранения:
# prompts/{domain}/{capability}/{role}/v{version}.md

# Пример промта:
"""
---
provider: openai
role: system
status: active
variables:
  - name: task_description
    type: string
    required: true
    description: "Описание задачи для анализа"
---

# Инструкции для агента анализа безопасности

Ты являешься экспертом в области безопасности кода...
"""
```

## Установка и начало работы

Для установки фреймворка:

```bash
# Клонировать репозиторий
git clone https://github.com/AlexEgorov85/Agent_code.git
cd Agent_code

# Создать виртуальное окружение
python -m venv venv
source venv/bin/activate  # или venv\Scripts\activate на Windows

# Установить зависимости
pip install -r requirements.txt
```

Простой пример использования:

```python
import asyncio
from application.factories.agent_factory import AgentFactory
from domain.value_objects.domain_type import DomainType

async def main():
    # Создать агента
    agent = await AgentFactory().create_agent(
        agent_type="composable",
        domain=DomainType.CODE_ANALYSIS
    )
    
    # Выполнить задачу
    result = await agent.execute_task(
        task_description="Проанализируй этот Python код на безопасность",
        context={
            "code": "def hello(): pass",
            "language": "python"
        }
    )
    
    print(f"Результат: {result}")

if __name__ == "__main__":
    asyncio.run(main())
```

## Безопасность

Фреймворк включает встроенные механизмы безопасности:

- **Валидация промтов**: Проверка на безопасность и корректность
- **Контроль доступа**: Ограничение доступа к ресурсам
- **Изоляция выполнения**: Предотвращение побочных эффектов
- **Фильтрация чувствительных данных**: Защита конфиденциальной информации
- **Обработка ошибок**: Надежная обработка исключений

## Расширяемость

Фреймворк легко расширяется через:

1. **Новые паттерны мышления**: Для новых стратегий решения задач
2. **Новые атомарные действия**: Для новых возможностей взаимодействия
3. **Новые домены**: Для новых областей применения
4. **Новые промты**: Для новых специфичных инструкций
5. **Новые инструменты и навыки**: Для интеграции с внешними системами
