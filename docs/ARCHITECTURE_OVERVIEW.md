## 🔍 Обзор

Agent_v5 — модульная платформа для автономных AI-агентов с поддержкой reasoning-циклов, планирования и выполнения задач.

### Назначение

- **Автономные агенты**: Выполнение сложных задач через цикл ReAct
- **Планирование**: Декомпозиция целей на последовательности действий
- **Интеграция**: Поддержка различных LLM-провайдеров и баз данных
- **Безопасность**: Изоляция между агентами, валидация контрактов

### Ключевые возможности

- ✅ **Reasoning-циклы**: ReAct-паттерн с валидацией действий
- ✅ **Планирование**: Иерархическая декомпозиция задач
- ✅ **Изоляция**: Каждый агент имеет собственный контекст и кэши
- ✅ **Версионирование**: Промпты, контракты и паттерны поведения
- ✅ **Структурированный вывод**: Нативная валидация провайдеров
- ✅ **Предзагрузка ресурсов**: 0 обращений к хранилищу во время выполнения

## 🏗️ Архитектурные принципы

### 1. Разделение слоёв

```
Infrastructure Layer (общий для всех агентов)
    ↓
Application Layer (изолирован на агента)
    ↓
Session Layer (данные сессии)
```

### 2. Декларация ≠ Данные ≠ Реализация

- **Декларация**: Манифесты компонентов в `data/manifests/`
- **Данные**: Промпты и контракты в `data/prompts/`, `data/contracts/`
- **Реализация**: Код компонентов в `core/`

### 3. Предзагрузка → Кэш → Выполнение

- Все ресурсы загружаются при инициализации
- Компоненты используют изолированные кэши
- Во время выполнения нет обращений к внешним хранилищам

## 📐 Слои системы

### Инфраструктурный слой

**Назначение**: Управление тяжёлыми ресурсами, общими для всех агентов

**Компоненты**:
- `InfrastructureContext` — общий контекст инфраструктуры
- `ProviderFactory` — фабрика провайдеров (LLM, БД)
- `ResourceRegistry` — реестр ресурсов
- `LifecycleManager` — управление жизненным циклом
- `EventBus` — шина событий системы

### Прикладной слой

**Назначение**: Изолированная логика конкретного агента

**Компоненты**:
- `ApplicationContext` — изолированный контекст агента
- `ComponentRegistry` — реестр компонентов агента
- `ComponentConfig` — конфигурация компонента
- `BaseService`, `BaseSkill`, `BaseTool` — базовые классы

### Слой сессии

**Назначение**: Хранение данных сессии агента

**Компоненты**:
- `SessionContext` — контекст сессии
- `DataContext` — данные сессии (истина)
- `StepContext` — шаги reasoning-цикла

## 🔗 Контексты

### InfrastructureContext

Общий для всех агентов контекст инфраструктуры.

```python
class InfrastructureContext:
    provider_factory: ProviderFactory
    resource_registry: ResourceRegistry
    lifecycle_manager: LifecycleManager
    event_bus: EventBus
    config: SystemConfig
```

**Особенности**:
- Создаётся один на приложение
- Содержит тяжёлые ресурсы (модели LLM, подключения к БД)

### ApplicationContext

Изолированный контекст каждого агента.

```python
class ApplicationContext:
    component_registry: ComponentRegistry
    infrastructure_context: InfrastructureContext  # read-only
    config: AppConfig
```

**Особенности**:
- Создаётся для каждого агента отдельно
- Содержит изолированные кэши промптов и контрактов

### SessionContext

Контекст сессии агента.

```python
class SessionContext:
    data_context: DataContext      # вся истина, сырые данные
    step_context: StepContext      # reasoning / шаги
    current_plan_item_id: str      # ссылка на текущий элемент плана
```

### Типы компонентов

| Тип | Базовый класс | Назначение | Примеры |
|-----|---------------|------------|---------|

## 🔄 Поток данных

### Инициализация приложения

1. Создание `InfrastructureContext`
2. Загрузка провайдеров и ресурсов
3. Создание `ApplicationContext` для агента
4. Инициализация компонентов с предзагрузкой

### Reasoning-цикл агента

1. `BehaviorManager.execute_pattern()`
2. `think()` → LLM генерирует мысль
3. `select_capability()` → выбор компонента
4. `execute(action)` → выполнение действия
5. Валидация контрактов
6. Запись в `SessionContext`

## 🔐 Безопасность

### Изоляция агентов

| Аспект | Реализация |
|--------|------------|
| **Кэши** | Изолированные на каждый агент |
| **Конфигурация** | ComponentConfig на компонент |
| **Провайдеры** | Общие, доступ через фабрику |
| **Данные сессии** | Полная изоляция SessionContext |

### Профили безопасности

| Профиль | Разрешённые версии | Побочные эффекты |
|---------|-------------------|------------------|
| **prod** | Только `active` | Разрешены |
| **dev** | `active`, `draft` | Разрешены |
| **sandbox** | Все | Заблокированы |
